日常使用

標記 → 訓練 → 推論

開①分類網站，Stage 1 蒐集資料、Stage 2 標記 fail pattern
跟以前完全一樣，純瀏覽器操作，不用裝任何東西。
Stage 2 勾選「排除 None」，匯出 TrainReady JSON
檔名長得像 StripFailPattern_Stage2_Dataset_TrainReady_YYYYMMDD.json，存到電腦裡記得的位置。
開②訓練主控台「訓練」分頁，上傳這份 JSON，按開始訓練
模型選 both 最方便比較，其他參數維持預設即可。跑完會看到每個模型的準確率、分類報告、混淆矩陣。
確認結果上方出現綠色「已同步到 Hub」標籤
代表這次訓練出的 .pt 已經自動存進③模型倉庫，就算 Space 之後睡眠重啟也不會遺失。想再三確認的話，直接開③的連結看檔案時間戳。
切到②「推論」分頁，上傳新的（未標記或要覆核的）Dataset JSON
選 mlp 或 cnn，按開始推論，就會列出每個 strip 的 AI 判斷結果、信心分數，可以跟人工標記或 Stage 2 規則式判斷對照。
這整段流程完全不需要碰本機的任何 Python / venv，從頭到尾都是瀏覽器操作。
要改程式碼的時候

Local 檔案 → 推上雲端

情境 A：只改①分類網站的 Stage 1/2 內容（index.html）

本機用任何文字編輯器改好 index.html
純文字檔，不需要裝任何工具。
開①的 Files 分頁，把新的 index.html 拖上去覆蓋舊檔
Static Space 不用等 build，存檔後幾秒內網站就是新版本。

情境 B：改②訓練/推論主控台的程式碼（app.py / train_core.py / predict_core.py / 網頁介面）

本機改好對應的檔案
要改網頁介面就改 templates/index.html，要改訓練邏輯就改 train_core.py，以此類推。
開②的 Files 分頁，把改過的檔案拖上去覆蓋
記得檔案要落在 repo 根目錄（跟 Dockerfile 同一層），不要連同資料夾一起拖上去。
等 Space 自動重新 build（Docker SDK，通常 1-2 分鐘）
畫面會顯示 Building → Starting → 正常運作。build log 有錯誤的話直接複製貼給 Claude 看。
卡住的時候

常見狀況速查

畫面顯示代表什麼怎麼做
Runtime error
Scheduling failure: not enough hardware capacity		 HF 免費硬體暫時排滿，跟你的程式碼無關 Settings 裡 Restart，不行就等 10-20 分鐘再試
Starting… / Building… 正常啟動中 等 1-2 分鐘，重新整理頁面
推論時顯示「找不到模型檔」 本機沒有、Hub 也抓不到 先去「訓練」分頁跑過一次訓練；再檢查 STRIP_MODEL_REPO_ID / HF_TOKEN 是否設定正確
訓練結果顯示橘色「僅存本機」 Hub 推送失敗（通常是 token 或 repo id 打錯/過期） 檢查②Settings → Variables and secrets 裡兩個值，必要時重新產生 token
Token 外流處理：如果不小心把 hf_... 開頭的 token 貼在截圖、聊天訊息、或任何別人看得到的地方， 去 huggingface.co/settings/tokens 把那個 token Revoke，重新建一個 Write token， 再回②的 Secret 把 HF_TOKEN 換成新的值。
