Strip Test Defect Fail Pattern 自動偵測模型

說明 Stage 1（資料蒐集 / Dataset Builder）與 Stage 2（Fail Pattern 歸類模型）如何串接， 將測試 log（VTH／VFSD／IPD-10V／IDSS1～IDSS3）轉換為可訓練、可推論的空間 fail pattern 分類系統。

1. 為什麼要分兩個 Stage

Strip test 的 fail pattern 和「測試項」高度相關 —— 同樣是 fail，發生在邊緣、單一 row、還是整片群聚，代表的製程異常完全不同。要讓 AI 判斷 pattern，必須先有「乾淨、結構化、有空間座標」的資料，這正是 Stage 1 要解決的問題；Stage 2 才是真正的分類模型。

  • Stage 1 — 資料蒐集：把原始 xls log 轉成「每個 strip 一筆」的結構化樣本（特徵向量 + 13×13 空間網格），並提供人工標記介面，產出可直接餵給模型的 dataset。
  • Stage 2 — Fail Pattern 歸類：拿 Stage 1 產出的 dataset 做訓練 / 推論，把每個 strip 分類成 Edge、Corner、Streak、Cluster、Random…等已知異常樣態，用於良率分析與製程異常告警。
兩個 Stage 是獨立可運作的模組：Stage 1 現在就能單獨使用（匯出 CSV/JSON 給工程師人工判讀）；Stage 2 之後可以隨時接上，不需要重新設計資料格式。

2. 整體架構

兩個 Stage 之間用一份共同的「Dataset JSON / CSV」規格銜接，這是整個系統唯一的耦合點。

STAGE 1 資料蒐集（Dataset Builder）

目的：把非結構化的測試機台 log，轉換成「每顆 die 有座標與 pass/fail」「每個 strip 有一組特徵向量」的乾淨資料集。

3.1　解析管線

1
讀取 LoggerData 分頁
用 SheetJS 解析 xls/xlsx，定位第 10 列表頭、第 4/5 列 Min/Max Limit。
2
限定判定測試項
只取 VTH、VFSD、IPD-10V、IDSS1、IDSS2、IDSS3 六欄；其餘（BVDSS/Delta3…）不參與 fail 判定。
3
Die 級 pass/fail 判定
每個測試項各自比對 Min/Max Limit（可用檔案內建值，也可手動覆寫），標記 fail 旗標。
4
還原空間座標
Log 內的 X 欄其實是 1~169 的序號，用 x=⌈X/13⌉，y=13-((X-1) mod 13) 還原成 13×13 實體網格；Remark 欄末 2 碼為 slot（strip）編號。
5
依 Lot + Slot 分組
每組即為一個 strip 樣本（一個 13×13 的 die 網格），是 Stage 2 分類的最小單位。
6
特徵工程
在每個 strip 上計算良率、各測試項 fail rate、空間分布特徵（下方 3.2）。
7
人工標記（可選）
工程師可在 heatmap 預覽介面手動標註 pattern 類別，作為 Stage 2 監督式學習的 ground truth。

3.2　每個 Strip 產出的特徵（Feature Vector）

Yield / Fail Count
總良率、fail die 數，作為基本篩選門檻。
各測試項 Fail Rate
VTH / VFSD / IPD-10V / IDSS1-3 各自的 fail count 與比例，判斷 Dominant Fail Item。
Edge / Interior Fail Ratio
邊界 die vs 內部 die 的 fail 比例 → 判斷 Edge-Ring 樣態。
Center Fail Ratio
中心區塊 fail 比例 → 判斷 Center 樣態。
Row / Column Streak
單一 row 或 column 的最大 fail 密度 → 判斷條狀（Streak）樣態。
Cluster Score
Fail die 中，有相鄰 fail die（8-neighbor）的比例 → 判斷群聚樣態。
Bounding-Box Density
Fail die 外接矩形的密度 → 判斷是否集中在小範圍。
Quadrant 分布
四象限 fail 數與主導象限 → 判斷 Corner 樣態。
這些特徵本身就是可解釋的判斷依據；Stage 2 無論用規則式或機器學習，輸入端都是這組特徵（或搭配原始 13×13 grid）。

STAGE 2 Fail Pattern 歸類模型

目的：拿 Stage 1 產出的 dataset，訓練一個能自動把每個 strip 分類到已知異常樣態的模型，並可對新 log 即時推論。

4.1　兩種可行的模型路線

路線 A · 規則式 / 統計式

Heuristic Classifier

  • 直接用 Stage 1 的特徵（edge ratio、streak、cluster score…）設定門檻值分類
  • 不需要標記資料就能先上線，作為 baseline
  • 可解釋性最高，工程師容易驗證與調整門檻
  • 缺點：門檻需人工調校，遇到混合型 pattern（如同時 Edge + Cluster）較難處理
路線 B · 機器學習

Feature-based / Image-based Classifier

  • Tabular 模型（如決策樹 / Random Forest / 小型 MLP）：輸入 Stage 1 的特徵向量
  • Image-based 模型（如小型 CNN）：直接輸入 13×13 grid（pass/fail/測試項別 one-hot），保留完整空間資訊
  • 需要 Stage 1 累積的人工標記樣本作為訓練資料
  • 優點：能學到規則式難以描述的混合樣態，且會隨資料量提升準確率
建議做法：先用路線 A 上線建立 baseline 並開始累積人工標記，樣本量足夠後（例如每類別 ≥ 30～50 個標記樣本）再導入路線 B，並用路線 A 的結果做交叉驗證。

4.2　訓練流程

1
彙整標記資料
收集多批 Stage 1 匯出的 Dataset JSON，篩出已標記（manualLabel 非空）的 strip 作為訓練集。
2
切分訓練 / 驗證集
依 Lot 或時間切分，避免同一 Lot 內的 strip 同時出現在訓練與驗證集（防止資料洩漏）。
3
特徵標準化
對 fail rate、streak、cluster score 等做標準化；類別不平衡（正常 strip 遠多於異常）需做加權或取樣調整。
4
模型訓練與選擇
比較路線 A（門檻）與路線 B（模型）在驗證集上的表現，以混淆矩陣、各類別 Precision/Recall 評估。
5
部署推論
新 log 進來後，重跑 Stage 1 相同的解析與特徵計算，輸出結果餵給 Stage 2 模型，得到 pattern 類別與信心分數。
6
回饋標記
工程師對模型判斷不同意時可修正標記，修正結果回流成新的訓練資料，形成持續改善迴圈。

4.3　推論輸出

對每個 strip，Stage 2 輸出：

  • Pattern 類別（如 Edge-Ring / Row-Streak / Cluster…）
  • 信心分數（模型對此分類的把握程度）
  • 對應 heatmap（沿用 Stage 1 的視覺化邏輯，直接標註判定依據，方便工程師覆核）

6. Stage 1 → Stage 2 資料交接格式

Stage 2 的所有輸入，都來自 Stage 1 匯出的三種檔案之一。

6.1　Strip-Level CSV（訓練用主要 Dataset，一列 = 一個 strip）

欄位說明
Lot / SlotLot 檔名、Slot（strip）編號
TotalDies / FailDies / Yield該 strip 的基本良率資訊
{ITEM}_FailCount / {ITEM}_FailRateVTH/VFSD/IPD-10V/IDSS1-3 各自的 fail 統計
DominantFailItemfail 數最多的測試項
EdgeFailRatio / InteriorFailRatio / CenterFailRatio空間分布特徵
MaxRowFailFrac / MaxColFailFrac條狀樣態偵測特徵
ClusterScore / BBoxDensity / BBoxW / BBoxH群聚樣態偵測特徵

6.2　Die-Level CSV（逐顆資料，供細部分析 / debug）

欄位說明
Lot / Slot / Serial / X / Y身分與座標
VTH / VFSD / IPD_10V / IDSS1 / IDSS2 / IDSS3實測值
{ITEM}_Fail個別測試項 pass/fail（0/1）

6.3　Dataset JSON（訓練 Image-based 模型 / 完整存檔用）

欄位說明

strips[].features同 Strip-Level CSV 的特徵物件
strips[].grid13×13 陣列，每格為 {s:"pass"|"fail"|"nodata", item, multi}，可直接轉成訓練影像
strips[].manualLabel人工標記類別
strips[].dies逐顆原始資料（與 Die-Level CSV 對應）

7. 延伸規劃

  • 累積跨批 Lot 的標記資料後，導入路線 B（ML / CNN）模型並持續迭代
  • 將 Dominant Fail Item + Pattern 類別組合，建立製程異常對照表（例如「IPD-10V + Edge-Ring」對應特定製程站點）
  • 視需要擴充判定測試項（目前限定 VTH/VFSD/IPD-10V/IDSS1-3，其餘測試項可依相同架構加入）
  • Stage 2 推論結果可回寫至 Stage 1 的 heatmap 介面，做「AI 判讀 vs 人工標記」的覆核與再訓練迴圈
