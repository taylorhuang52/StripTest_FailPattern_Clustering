累積資料集總覽

模型輸入特徵（每個 strip → 19 維特徵向量）

空間分布特徵（8 維，來自 Stage1 Strip-Level 計算）： 1-Yield（總 fail 率）、EdgeFailRatio、InteriorFailRatio、CenterFailRatio、MaxRowFailFrac、MaxColFailFrac、ClusterScore（8-neighbor 群聚度）、BBoxDensity（fail 外接框密度）。
測試項別 fail rate（11 維）： VTH／GET_VTH_LO／GET_VTH_HI／VFSD／BVDSS2／BVDSS1／DELTA3／IPD_10V／IDSS1／IDSS2／IDSS3 各自的 fail 率（若某批資料未涵蓋該測試項，則補 0）。
訓練 / 推論前皆會用目前累積資料集重新做 Min-Max Normalize，並把 Scaler 隨模型權重一起匯出，確保之後匯入推論時使用同一組尺度。

歸類模型

不需訓練資料，依 EdgeRatio / RowColStreak / ClusterScore / Quadrant 等特徵套用工程經驗法則，做為冷啟動基準（Stage1 剛開始累積、標記樣本還很少時可先用這個）。
以「已標記」樣本為訓練集，用正規化後的特徵向量做最近鄰投票。資料持續累積時完全不用重新訓練，新標記樣本立即生效——適合標記樣本還不多、類別會持續增加的早期階段。
多類別 Softmax 迴歸（線性分類器），以梯度下降訓練。支援「Fine-tune / 增量訓練」：每次點擊會從目前記憶體中的權重繼續訓練（warm start），新出現的類別會自動加入權重矩陣，不會打掉舊類別已學到的權重。資料量成長到中大型規模、且類別已相對穩定時，這個模型通常比 k-NN 更穩定、預測更快。
在同一組已標記樣本上比較三種模型的準確率（Rule-based：全量比對；k-NN：LOOCV；Softmax：目前權重在 held-out 驗證集上的準確率，若尚未訓練請先到③訓練）。

推論 & 人工校正（Active-Learning 循環）

選擇上方任一已「訓練/套用」的模型作為推論來源，對每個 strip 產生建議標籤與信心值；工程師確認或修正後按「採用」即寫回 ManualLabel，成為下一輪 Fine-tune 的新訓練資料——這就是「持續累積 + fine-tune」的完整循環。

Strip Test Fail Pattern AI — 架構總覽 & 使用說明

Stage 1（資料蒐集）→ Stage 2（歸類模型 / 累積 / Fine-tune）的完整流程，以及兩者之間的資料交換格式。

整體流程

STAGE 1 · Dataset Builder
  • 上傳 Strip Test .xls log（可多檔 / 多 lot）
  • 依「判定測試項設定」重算每顆 die 的 Pass/Fail
  • 依 Remark 還原 Slot，並依 GridSide 還原 2D 座標
  • 計算 Strip-level 空間特徵（Edge/Center/Cluster/Streak…）
  • Bin 欄位資料檢核，抓漏抓的測試項
  • yield>95% 自動預標 "None / No Fail"，其餘人工在 Heatmap 上快速標記
輸出：StripFailPattern_Dataset_*.json
→
STAGE 2 · 歸類模型
  • 匯入 / 累積多批 Stage1 Dataset JSON（去重、保留既有標記）
  • 19 維特徵向量 + Min-Max Normalize
  • Rule-based（冷啟動）／k-NN（即時更新）／Softmax（可 Fine-tune）
  • LOOCV / Train-Val 準確率、Confusion Matrix
  • 對未標記 strip 產生建議標籤 + 信心值
  • 工程師確認/修正 → 寫回標記 → 下一輪 Fine-tune
輸出：MasterDataset.json / 模型權重 / 預測 CSV / Model Card
⟲ Stage2 的標記校正結果可匯出、再匯入回 Stage1 的資料（或直接留在 Stage2 累積），持續擴大訓練集 → 下次 Fine-tune 準確率持續提升。

資料交換格式（JSON）

Stage1 匯出、Stage2 匯入的是同一份 schema，關鍵欄位：
key= Lot__slotNN，跨批合併去重用
featuresstrip-level 空間特徵（yield、edgeFailRatio、clusterScore…）+ 各測試項 fail rate
manualLabel人工（或自動預標）的 pattern 標籤，Stage2 訓練用的 ground truth
autoLabeledtrue = 由 yield>95% 規則自動預標、尚未人工複核
grid已算好的 2D pass/fail/item 網格，Stage2 直接拿來畫 Heatmap，不用重算
diesdie-level 明細（座標、量測值、各項目 Pass/Fail、原始 Bin）

Stage 2 模型選擇邏輯

階段建議模型原因
標記樣本很少（< 30 筆 / 類別不足）Rule-based不需訓練資料，工程經驗直接編碼，先求「堪用」
標記樣本持續累積中（幾十～數百筆）k-NN新標記立即生效、不用重訓，適合標記還在快速變動的階段
類別已相對穩定、樣本量變大Softmax 迴歸線性分類器，訓練/推論皆快，支援 warm-start 增量 Fine-tune
未來：樣本量達數千筆以上CNN（Stage 3 構想）直接吃 2D fail map 影像，不需手工特徵；樣本量不足前效益有限，暫不建置

使用步驟

  1. Stage 1：上傳當批 .xls，確認「判定測試項設定」涵蓋所有測試項，按「開始處理」。留意 Bin 欄位資料檢核是否出現 ⚠ 不一致警告。
  2. 在 Strip 列表中檢視 Heatmap，yield>95% 的 strip 已預標 "None / No Fail"（灰底 auto 標籤），只需快速確認；其餘 strip 依實際圖形挑選 Pattern Label。
  3. 按「匯出 Dataset JSON」，得到本批的 Stage1 Dataset。
  4. 切到 Stage 2，把該 JSON（可一次丟多批）拖進「匯入 / 累積 Dataset」。
  5. 視累積量選擇模型：樣本少先用①Rule-based 或②k-NN 驗證；樣本夠了用③Softmax「Train / Fine-tune」（會自動 warm-start）。
  6. 到⑤「推論 & 人工校正」，選一個模型產生建議標籤，逐筆確認或按「採用」寫回 ManualLabel。
  7. 匯出 Master Dataset JSON 備份（含所有標記與模型權重），下次開啟 Stage 2 直接匯入即可從累積進度和權重繼續，不用從頭再標一次。
